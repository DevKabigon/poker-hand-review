# Design Fix 01 - Concept Lock

작성일: 2026-02-28  
범위: Phase 1 / Step 1 (디자인 컨셉 확정)

## 요청 해석

- 목표: "플레이풀하지만 게임 UI처럼 보이지 않는" 포커 핸드 리뷰 서비스
- 필수 조건:
  - 신뢰감 있는 SaaS 톤 유지
  - 버튼은 눌리는 감각(press-down micro interaction) 적용
  - 다크/라이트 모두 품질 유지
  - 랜딩 스크롤 길이는 짧고 메시지는 명확하게

## 확정 컨셉

- 컨셉명: `Soft Tactile Analyst`
- 키워드: `정제됨`, `입체감`, `빠른 피드백`, `과장 없는 플레이풀`
- 배제 요소:
  - 과한 게임풍 아이콘/네온/글로우
  - 카드 테마 장식 남발
  - 과도한 애니메이션

## 비주얼 원칙

1. 색
- 베이스: soft white / soft black (순백/순흑 금지)
- 브랜드 컬러는 강조용으로만 제한 사용
  - Primary: `#00bc7d`
  - Accent: `#8c52ff`

2. 형태
- 표면은 `flat`이 아니라 `soft elevation` 중심
- 버튼/카드 라운드는 유지하되 장난감 느낌이 아닌 밀도 높은 그림자 사용

3. 타이포
- 본문은 가독성 우선
- 헤드라인은 약간의 개성을 주되 과한 디스플레이 폰트는 지양
- 모바일 기준 `clamp()` 스케일로 리듬 통일

## 인터랙션 원칙

1. Pressable Button (핵심)
- 기본: 하단 그림자 + 약한 상단 하이라이트
- Hover: 살짝 상승
- Active: 아래로 눌리며 그림자 깊이 축소
- Focus: ring 명확화

2. 모션
- 페이지 진입 시 1회성 짧은 reveal
- 버튼/카드는 120~180ms 내 짧은 피드백
- 스크롤 인터랙션은 최소화

## 페이지 톤 가이드

1. 랜딩
- 플로팅 네브 + 짧은 Hero + 핵심 기능 요약 + CTA + 컴팩트 푸터
- "많이 스크롤해야 하는 랜딩" 금지

2. 로그인/회원가입
- 동일한 auth shell로 통일
- 입력/버튼의 tactile 피드백 강조
- OAuth는 자리만 유지하고 연결은 사용자 작업 범위

## 아키텍처 가이드 (적용 스킬 반영)

- `vercel-composition-patterns`
  - boolean prop 확장 대신 명시적 variant/compound 구조 사용
- `vercel-react-best-practices`
  - 랜딩은 서버 컴포넌트 중심 유지
  - 클라이언트 상호작용은 최소한으로 분리
